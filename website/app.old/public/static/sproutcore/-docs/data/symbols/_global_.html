Method Details

beginEditing()

Call this method to make the inline editor begin editing for your view.

-   Boolean  beginEditing (options)

Return Value

Boolean - 

Discussion

If the inline editor is already being used for another value it will try to dismiss itself from the other editor and attach itself to the new view instead. If this process fails for some reason (for example if the other view did not allow the view to end editing) then this method will return false.

You should pass a set of options that at least includes the target view. See class definition for options.

commitEditing()

Save the current value of the inline editor and exit edit mode.

-   Boolean  commitEditing ()

Return Value

Boolean - 

Discussion

If the inline editor is being used it will try to end the editing and close. If the inline editor could not end for some reason (for example if the delegate did not allow the editing to end) then this method will return NO.

deliverNotifications()

Tries to flush observer notifications.

-   void  deliverNotifications ()

Return Value

void - 

Discussion

The idea is that this will continue to remove observers until all of the notifications have flushed or until the max execution time has passed. Called automatically from endRunLoop().

The way this works, it does not unshift pending notifications but instead just sets them to null until the entire queue is flushed. Then it will reset the queue to a new array. This is faster than shifting().

discardEditing()

Discard the current value of the inline editor and exit edit mode.

-   Boolean  discardEditing ()

Return Value

Boolean - 

Discussion

If the inline editor is in use, this method will try to end the editing, restoring the original value of the target view. If the inline editor could not end for some reason (for example if the delegate did not allow editing to end) then this method will return NO.
