SC.Controller

The controller base class provides some common functions you will need for controllers in your applications, especially related to maintaining an editing context.

In general you will not use this class, but you can use a subclass such as ObjectController, CollectionController, or ArrayController.

EDITING CONTEXTS

One major function of a controller is to mediate between changes in the UI and changes in the model. In particular, you usually do not want changes you make in the UI to be applied to a model object directly. Instead, you often will want to collect changes to an object and then apply them only when the user is ready to commit their changes.

The editing contact support in the controller class will help you provide this capability.

Property Details

commitChangesImmediately

If this is false, then the controller will only commit changes when you explicitly call commitChanges.

-   Boolean commitChangesImmediately

Discussion

Otherwise it will commit them immediately. You usually want this set to false. It is initially set to true for compatibility.

context

This is the controller's parent controller usually.

-   SC.Controller context

Discussion

The controller will notify this controller when its changes are committed or discarded.

hasChanges

The controller will set this property to true whenever there are changes that need to be committed.

-   Boolean hasChanges

Discussion

This property is true whenever the controller itself has uncommitted changes or when any dependent editors have uncommitted changes. In your own subclass, call this.objectDidChange(this) to register changes.
